- ✅ Iter 29 (PR #1385 → sprint): iOS Safari focus-zoom prevention. iOS auto-zooms the layout when any `<input>` / `<textarea>` / `<select>` with computed `font-size < 16px` is focused, and doesn't reset on blur — a real UX bug for any of the 10 codebase sites that override the shadcn `text-base md:text-sm` default to `text-xs` / `text-sm` for dense identifier displays (cron expression, slug, runtime path, API key paste/display, household-copy id, rule DSL JSON, action-row comment, prompts-builder textarea, etc.). Plus the `CommandInput` shadcn primitive renders `text-sm` with no mobile fallback, affecting all 7 of its consumers (cmdk searches in cron-field, transactions-toolbar, category-form, category-command, command-palette, icon-picker, tag-command). Fix: one global rule in `globals.css` — `@media (pointer: coarse) { input, textarea, select { font-size: 16px !important } }` (excluding `type=checkbox/radio/file/range/color`). Locks the zoom-prevention requirement at the CSS-base level so future contributors don't need to remember the iOS quirk per-call-site, and the call-site overrides keep their compact density on desktop. New Playwright regression test `e2e/ios-input-zoom.spec.ts` injects a probe input/textarea/select with `text-xs` and asserts ≥16px across all 4 webkit mobile projects. Confirmed clean on mobile-sweep.
- ✅ Iter 30 (PR #1386 → sprint): overscroll-behavior audit on Sheet/Dialog/route scroll containers. iOS Safari's pull-to-refresh and rubber-band overscroll leak through scrollable bodies into parent contexts unless `overscroll-behavior: contain` is set. Found 15 unprotected sites across 9 files: all 6 Sheet bodies (transcript-sheet, reauth-sheet, link-account-sheet, connect-bank-sheet, shortcut-sheet, csv-import-form's 2 panes + table preview), 3 sites inside the 2 Dialogs in prompts.build.tsx (AddBlockMenu rail + content + PreviewDialog `<pre>`), the 2 settings-shell scroll panes (mobile pill rail + content), the error-page stack trace `<pre>`, plus the transcript-sheet inner error log `<pre>`. Fix: applied `overscroll-contain [-webkit-overflow-scrolling:touch]` to each (matches the convention established by `components/ui/table.tsx` in iter 22 and the v2-frontend rules' horizontal-scroll pattern). shadcn `Sheet`/`Dialog`/`AlertDialog` primitives themselves remain untouched (per rules) — the consumer-side fix is sufficient because the scroll-handler always lives on a consumer-owned wrapper inside the SheetContent/DialogContent body. Mobile-sweep clean, no overflow regressions.
- ✅ Iter 31 (PR #1387 → sprint): icon-only button accessible-name audit. Comprehensive sweep across `src/` for raw `<button>`, shadcn `<Button>`, `<Link>`/`<a>`, and `<Button asChild>` patterns — every interactive icon-only element should expose an accessible name for VoiceOver/NVDA on iOS (WCAG 4.1.2). Three real fixes found: (1) `icon-picker.tsx:191` IconTile relied on `title={name}` alone — added `aria-label={name}` + `aria-pressed={selected}` for proper screen-reader announcement + toggle-state semantics; (2)+(3) `rule-form.tsx` AND/OR + STAGE_PRESETS segmented-control toggles had no pressed-state semantics — added `aria-pressed={...}` so VoiceOver announces "AND, selected" instead of just "AND, button". The broader audit confirmed the codebase has strong aria-label discipline: every `size="icon"` shadcn Button instance carries `aria-label`, decorative SVGs (search-input glyph, category-picker overlay) use `aria-hidden`, tag-chip × buttons and color-picker swatches all carry interpolated labels (`Remove ${tag}`, `Pick ${color}`), expand/collapse triggers use `aria-expanded` correctly. No further fixes needed in this dimension.
- ✅ Iter 32 (PR pending → sprint): `autoComplete` attribute audit. iOS Safari (and every other browser) decides whether to offer autofill, and what to offer, based on the `autoComplete` attribute on each `<input>`. The codebase already had it right on every security-sensitive field (login `username` + `current-password`, password-change `new-password`, provider client-id/secret) — but 23 other inputs across 17 files were missing the attribute, so iOS happily suggested "John Smith" for slug fields, contact emails for API key names, etc. Three identifying additions (`autoComplete="name"` on the household-member name, `autoComplete="email"` on the optional member email, `autoComplete="username"` on the create-login email so iOS knows to save the saved-credential pair) plus 20 explicit `autoComplete="off"` opt-outs on technical fields (slug, cron, runtime path, API key name, condition values, search boxes, numeric filters, account display name, readonly token displays). Bonus: added `autoComplete="off"` to the `SearchInput` primitive itself so future search consumers get it automatically; promoted the prompts-build dialog search to the full SearchInput-style ergonomics (type=search, inputMode, enterKeyHint, autoCorrect, autoCapitalize, spellCheck). Mobile-sweep clean.
